# 02. 던전/스테이지 템플릿

---

## 세계관 맥락

- **대륙/국가 내부** = 안전지대 (전투 없음)
- **대륙↔대륙, 국가↔국가 사이** = 오픈 필드 (적 출현) — 별도 시스템
- **던전** = 국가 내부의 **랜드마크**. 랜드마크 진입 → 던전 씬. 최상층에 보스.
- 이 설계도는 **던전(층–미션–보스 구조)** 에만 집중한다.

---

## 1. 목표 / 완료 정의

- 랜드마크 진입 → 던전 씬 로드 → 층(룸) 진행 → 미션 수행 → 최종 층 보스 → 보상 지급 → 복귀의 **한 사이클이 동작**한다.
- `DungeonDef` / `FloorDef` / `MissionDef` 데이터만 추가하면 새 던전을 만들 수 있는 **템플릿**이다.
- 체크포인트 도달 기록, 전멸 추방, 재입장, 진행 영구 기록이 모두 동작한다.

---

## 2. 현재 상태 & 재사용 자산

| 자산 | 경로 | 재사용 방식 |
|------|------|-----------|
| 적 레지스트리 | `Scripts/Combat/Enemy.cs` | `Enemy.All.Count == 0` 으로 몬스터 소탕 완료 감지 |
| 적 스포너 | `Scripts/Combat/EnemySpawner.cs` | 몬스터 소탕 미션에서 적 스폰 |
| 데미지 계산 | `Scripts/Combat/CombatMath.cs` | 변경 없이 사용 |
| 상호작용 | `Scripts/Interaction/Interactable.cs` | 랜드마크 입구, 계단/포탈, 기믹 장치, 아이템 오브젝트에 부착 |
| 골드·재화 지급 | `MetaState.Wallet.Add(CurrencyKind, amount)` | 보상 지급 |
| 연구자원 지급 | `MetaState.StudyMaterials.Add(Continent, amount)` | 보상 지급 |

## 3. 아키텍처

### 데이터 (ScriptableObject)

```
DungeonDef
├── displayName          : string       — 표시 이름 ("어둠의 탑")
├── continent            : Continent    — 소속 대륙
├── recommendedLevel     : int          — 권장 레벨 (입장 조건 표시용)
└── floors               : FloorDef[]  — 1층~보스 층 순서

FloorDef
├── floorNumber          : int          — 층 번호 (1-based)
├── roomPrefab           : GameObject   — 이 층의 룸 프리팹 (씬 내 비활성 오브젝트)
├── isCheckpoint         : bool         — 체크포인트 여부
├── isBoss               : bool         — 보스 층 여부
└── missions             : MissionDef[] — 이 층의 미션 목록

MissionDef
├── missionType          : MissionType  — 몬스터소탕 / 미로찾기 / 수집운반 / 기믹퍼즐 / 보스처치
├── isRequired           : bool         — true = 필수(다음 층 개방 조건), false = 선택(추가 보상)
├── objectiveCount       : int          — 목표 수량 (수집 개수, 소탕 수 등 타입에 따라 해석)
├── missionReward        : RewardDef    — 미션 완료 보상
└── description          : string       — UI 표시용 설명
```

**RewardDef (Serializable 클래스)**
- `gold`, `paper`, `focus`, `fragment` (각 재화 양)
- `expReward` (캐릭터 경험치)
- `studyContinent` / `studyAmount` (연구자원 — 해당 대륙 과목)
- `isFirstClearBonus : bool` — 이 보상이 최초 클리어 전용인지 여부

**보상 단위 정리**
| 단위 | 보상 정의 위치 |
|------|-------------|
| 미션 완료 | `MissionDef.missionReward` |
| 층 클리어 (필수 미션 전부) | `FloorDef.floorClearReward` |
| 던전 클리어 (보스 처치) | `DungeonDef.clearReward` (최초·반복 구분) |

---

### 런타임 (MonoBehaviour, 던전 씬)

**DungeonRunner** — 던전 씬의 빈 오브젝트에 부착. DungeonDef를 인스펙터에서 연결.

```
흐름
DungeonRunner.Start()
  → CheckpointService.Init(dungeon) — 저장된 체크포인트 로드
  → LoadFloor(진입 지점 층)         — 해당 룸 프리팹 활성화

[층 진행 중]
  MissionController들이 미션 완료 → DungeonRunner.OnMissionComplete(mission) 호출
    → 미션 보상 지급
    → 해당 층 필수 미션 전부 완료? → OnFloorCleared()
        → 층 클리어 보상 지급
        → 체크포인트 층이면 CheckpointService.RecordCheckpoint(floor)
        → 다음 층 계단/포탈 개방 (Interactable 활성화)

[다음 층 계단/포탈 E키]
  → DungeonRunner.AdvanceFloor(nextFloor)
      → 현재 룸 비활성화, 다음 룸 활성화 (씬 재로드 없음)

[보스 층 진입]
  → 보스 미션(BossController) 시작

[보스 처치 — 던전 클리어]
  → 클리어 보상 지급 (최초/반복 구분)
  → MetaSaveService.Save()
  → 연출 후 씬 복귀 (01번 전환 인프라 사용)

[파티 전멸]
  → PlayerDeath 이벤트 → DungeonRunner.OnPartyWiped()
      → MetaSaveService.Save() (체크포인트 기록 보존)
      → 던전 씬 → 안전지대(국가 내부) 씬 복귀
      → 다음 입장 시 CheckpointService.GetRespawnFloor()로 재시작 층 결정
```

---

**MissionController (타입별 — 각 룸 프리팹 안에 배치)**

| 타입 | 완료 감지 방식 |
|------|-------------|
| 몬스터 소탕 | `Enemy.All.Count == 0` (룸 안의 적 전멸) |
| 미로 찾기 | 목표 지점 Trigger에 플레이어 도달 (`OnTriggerEnter`) |
| 수집/운반 | 아이템 오브젝트(`Interactable`)를 N개 획득 후 반납 지점 도달 |
| 기믹/퍼즐 | 장치들의 `isActivated` 플래그 전부 true |
| 보스 처치 | 보스 Enemy HP = 0 |

각 MissionController는 완료 시 `DungeonRunner.OnMissionComplete(this.missionDef)`를 호출.

---

**CheckpointService** (static, 씬 전환 후에도 유지)

- `Init(DungeonDef)` — PlayFab에서 이 던전의 `DungeonProgress` 로드
- `RecordCheckpoint(FloorDef)` — 현재 최고 체크포인트 업데이트 + 즉시 저장
- `GetRespawnFloor()` — 마지막 체크포인트 층 반환 (없으면 1층)
- `CanTeleportTo(FloorDef)` — 도달한 체크포인트 층끼리만 true
- 도달한 체크포인트 층 목록 → 던전 HUD에서 순간이동 UI로 표시

---

**층 이동 규칙 (이동의 자유)**

| 이동 종류 | 조건 | 방식 |
|----------|------|------|
| 위아래 도보 | 해당 층을 클리어한 경우 | 계단/포탈 E키 → `AdvanceFloor` |
| 체크포인트 순간이동 | 도달한 체크포인트 층끼리 | DungeonHud 지도 UI에서 선택 → `TeleportToCheckpoint` |

---

**저장: DungeonProgress**

```
DungeonProgress (MetaState에 등록)
├── string dungeonId              — DungeonDef의 고유 ID
├── int    highestCheckpointIndex — 이 던전에서 도달한 최고 체크포인트 층 번호
├── bool   isCleared              — 보스 처치 완료 여부
└── bool   isFirstClearDone       — 최초 클리어 완료 여부 (최초 클리어 보상 지급 여부)
```

`MetaState.DungeonProgresses` (Dictionary\<string, DungeonProgress\>) 로 관리.
`MetaSaveService`에 `KEY_DUNGEON = "dungeonProgress"` 추가, 기존 Save/Load dict 패턴 그대로 확장.

---

### 씬 구조 (던전 씬 오브젝트 계층)

```
[던전 씬]
- DungeonManager
    - DungeonRunner       ← DungeonDef 연결
- Rooms                   ← 모든 FloorDef 룸 프리팹이 여기 자식으로, 기본 전부 비활성
    - Floor_01_Room       (isCheckpoint)
    - Floor_02_Room
    - Floor_03_Room       (isCheckpoint)
    - ...
    - Floor_Boss_Room     (isBoss)
- Player                  ← 01번 전환 인프라로 스폰 위치 결정
- HUD Canvas
    - CombatHud           ← HP/MP (기존 컴포넌트 재사용)
    - DungeonHud          ← 층 이름, 미션 목표 목록, 체크포인트 지도
```

## 6. 엣지 케이스 / 주의점

| 상황 | 처리 방식 |
|------|----------|
| 룸 전환 중 살아있는 적 | 룸 비활성화 시 `Enemy.All`에서 해당 룸 적을 모두 제거 (룸 프리팹이 비활성되면 Enemy.OnDisable에서 자동 제거됨) |
| 체크포인트 순간이동 도착 시 적·미션 상태 | 도착 층의 룸이 이미 클리어됐으면 적 없음. 미클리어 체크포인트 층에 이동 시 적·미션 상태 리셋(초기화) |
| 미클리어 층 도보 시도 | 계단/포탈 Interactable은 이전 층 필수 미션 완료 전까지 비활성 유지 |
| 보상 중복 지급 | 미션·층·던전 클리어 플래그(`_granted` bool)로 1회만 지급 보장 |
| 최초 클리어 보상 재지급 | `DungeonProgress.isFirstClearDone` 플래그로 방지 |
| 던전 클리어 후 재도전 | 재입장 가능. `isFirstClearDone` true이므로 일반 클리어 보상만 지급. 체크포인트는 유지되어 원하는 층 선택 가능 |
| PlayFab 저장 실패 (전멸 시) | 실패 로그 후 복귀 진행. 체크포인트 기록이 로컬 메모리에는 있으므로 세션 내 재시작은 정상 동작 |
| DungeonDef 미연결 | `DungeonRunner.Start()`에서 null 체크 → 경고 로그 + 즉시 복귀 씬 전환 |

## 7. 검증 방법

### 최소 검증 세트
테스트용 던전 1개 제작: **3층 (1층–체크포인트–보스 층)**, 미션 타입 4종 각 1개 포함.

| 검증 항목 | 기대 결과 |
