# 90. 관리자 시스템

---

## 1. 목표 / 완료 정의

- 게임 내 설정 메뉴에서 비밀 키를 입력해 관리자 인증이 된다.
- 인증 후 AdminPanel이 4탭(커맨드/캐릭터/플레이어/문제)으로 열린다.
- 웹(WebGL) 빌드에서도 설정 메뉴를 통해 관리자 패널에 접근 가능하다.
- 다른 플레이어 계정 조회·수정 및 문제 은행 관리가 CloudScript를 통해 인게임에서 가능하다.
- ProblemDatabase가 PlayFab TitleData로 이전되어 재빌드 없이 문제 추가·수정·삭제가 된다.

---

## 3. 아키텍처

### 3-0. 관리자 인증 흐름

```
설정 메뉴 → "관리자 로그인" 버튼 → AdminLoginPanel 열림
  → 비밀 키 입력 (TMP_InputField, 비밀번호 마스킹)
  → "확인" 버튼 → PlayFabCloudScript.ExecuteFunction("VerifyAdminKey", { key })
      CloudScript 내부:
        if (key == SECRET_ADMIN_KEY)   ← CloudScript에만 존재, 클라이언트에 없음
          return { success: true }
        else
          return { success: false }
  → 성공: MetaState.IsAdmin = true, AdminLoginPanel 닫힘, AdminPanel 열림
  → 실패: "잘못된 키입니다" 메시지
```

- 비밀 키는 **CloudScript 코드 안에만** 존재. 클라이언트 빌드에 없음.
- 인증은 **세션 메모리에만** 유지 (`MetaState.IsAdmin = true`). 게임 재시작 시 초기화.
- PlayFab ReadOnlyData 기록 없음 → 계정에 흔적 안 남음.

### 3-1. 패널 구조

```
AdminPanel (Canvas)
├── [탭 바]  커맨드 | 캐릭터 | 플레이어 | 문제
├── [A] CommandTab
├── [B] CharacterTab
├── [C] PlayerTab
└── [D] ProblemTab
```

탭 전환 시 해당 탭 GameObject만 SetActive(true), 나머지 false.

### 3-2. A. 커맨드 탭 (CommandTab)

본인 계정 치트 전용. 모두 로컬 처리 (CloudScript 불필요).

**재화 지급**
```
[종류 드롭다운: 골드/논문/집중력/조각/결정(종류 추가 드롭다운)]
[금액 입력 필드 (int, 기본값 1000)]
[지급] 버튼 → Wallet.Add(kind, amount) → Save
```
- 기존 고정 버튼(+골드 1000 등)은 제거하고 위 입력형으로 통합.
- 결정(Crystal)은 종류 드롭다운을 추가로 표시.

**레벨/스탯 조작**
```
[캐릭터 드롭다운: 보유 캐릭터 목록]
[대상: 레벨 | 스탯 항목(STR/INT 등) 드롭다운]
[값 입력 필드]
[적용] → CombatCharacter 해당 필드 강제 세팅 → Save
```

**퀘스트 강제 완료**
```
[퀘스트 ID 입력 필드]
[강제 완료] → QuestService.ForceComplete(id) → Save
```
- 퀘스트 시스템(05번)이 구현된 후 연동. 미구현 시 "퀘스트 시스템 미구현" 안내.

**가챠 디버그**
- 기존 롤×1, 롤×10 유지. 논문 자동 보충 방식 유지.

**계정**
- 기존 초기화 / 강제 저장 / 강제 불러오기 유지.

---

### 3-3. B. 캐릭터 탭 (CharacterTab)

본인 계정의 캐릭터 보유 현황 확인 및 지급/회수.

**UI 구조**
```
[검색 입력 필드]  →  입력 시 아래 목록 필터링
[전체 지급] 버튼

[스크롤 목록]
  각 행: [초상화 아이콘] [이름] [희귀도] [보유 여부 ✓/─] [지급/회수 버튼]
```

**동작**
- 탭 열릴 때 `CharacterDatabase.All`로 전체 목록 생성.
- `MetaState.Roster.Has(id)` 로 보유 여부 판단 → ✓ 또는 ─ 표시.
- [지급] 클릭 → `Roster.Add(id)` → 행 갱신 + Save.
- [회수] 클릭 → `Roster.Remove(id)` → 행 갱신 + Save.
- 검색 필드 입력 시 이름 부분 일치로 목록 필터링 (대소문자 무시).

---

### 3-4. C. 플레이어 탭 (PlayerTab) — CloudScript

다른 플레이어 계정 관리. 모든 작업은 CloudScript를 통해 PlayFab Admin API 사용.

**UI 구조**
```
[플레이어 검색: DisplayName 또는 PlayFabId 입력]  [검색]
[목록: PlayFabId | 표시이름 | 마지막 로그인 | 선택 라디오]

선택된 플레이어 대상 작업:
  ─ 캐릭터 지급/회수: [캐릭터 드롭다운] [지급] / [회수]
  ─ 재화 지급:       [종류 드롭다운] [금액 입력] [지급]
  ─ 계정 초기화:     [초기화] (확인 다이얼로그 필수)
  ─ 경고 메시지:     [메시지 입력] [발송]
```

**CloudScript 함수 목록**

| 함수명 | 인자 | 동작 |
|--------|------|------|
| `AdminGetPlayers` | `{ query: string }` | DisplayName/Id 검색 → 목록 반환 |
| `AdminGiveCharacter` | `{ targetId, charId }` | 대상 계정 Roster에 캐릭터 추가 |
| `AdminRevokeCharacter` | `{ targetId, charId }` | 대상 계정 Roster에서 캐릭터 제거 |
| `AdminGiveCurrency` | `{ targetId, kind, amount }` | 대상 계정 Wallet에 재화 지급 |
| `AdminResetAccount` | `{ targetId }` | 대상 계정 UserData 초기화 |
| `AdminSendWarning` | `{ targetId, message }` | 대상 계정 메시지 큐에 경고 저장 |

- 경고 메시지: PlayFab UserData의 `KEY_WARNINGS` 키에 JSON 배열로 저장. 해당 플레이어 로그인 시 `MetaSaveService.Load`에서 읽어 팝업 표시.
- 모든 CloudScript 함수는 호출자가 `MetaState.IsAdmin == true`일 때만 호출 가능하도록 클라이언트에서 사전 체크 (추가로 CloudScript 내에서도 호출자 PlayFabId 기반 이중 검증 가능).

### 3-5. D. 문제 탭 (ProblemTab) — CloudScript

문제 은행을 PlayFab TitleData에서 관리. ScriptableObject 재빌드 없이 인게임 수정.

**TitleData 구조**

```
TitleData 키: "problems_{characterId}"
값 (JSON):
{
  "levelup": [
    { "id": "p001", "text": "...", "choices": ["A","B","C","D"], "answer": 0, "difficulty": "Normal" },
    ...
  ],
  "skills": [
    {
      "skillId": "skill_fire",
      "problems": [
        { "id": "p010", "text": "...", "choices": [...], "answer": 2 }
      ]
    },
    ...
  ]
}
```

- 키: `problems_{characterId}` (예: `problems_maris`, `problems_zeno`)
- 레벨업 문제: 캐릭터마다 별도 풀, `difficulty` 필드로 Easy/Normal/Hard 구분.
- 스킬 연구 문제: `skillId`와 1:1 매핑.

**UI 구조**
```
[캐릭터 드롭다운]  [레벨업 / 스킬연구 탭]

레벨업 탭:
  [난이도 필터: All/Easy/Normal/Hard]
  [문제 목록: id | 미리보기(앞 30자) | 난이도]
  [+ 추가] [편집] [삭제]

스킬연구 탭:
  [스킬 드롭다운]
  [해당 스킬의 문제 목록]
  [+ 추가] [편집] [삭제]

문제 편집 팝업:
  문제 텍스트 / 선택지 A-D / 정답 인덱스 / 난이도
  [저장] → CloudScript.AdminUpdateProblem 호출
```

**CloudScript 함수 목록**

| 함수명 | 인자 | 동작 |
|--------|------|------|
| `AdminGetProblems` | `{ characterId }` | TitleData `problems_{characterId}` 반환 |
| `AdminUpdateProblem` | `{ characterId, type, problem }` | 문제 추가 또는 수정 (id 기준) |
| `AdminDeleteProblem` | `{ characterId, type, problemId, skillId? }` | 문제 삭제 |

**런타임 문제 로드 변경**

기존: `Resources.Load<ProblemDatabase>("ProblemDatabase")`  
변경: `PlayFabClientAPI.GetTitleData(keys: ["problems_{charId}"])` → JSON 파싱 → 메모리 캐시

- `ProblemService` (신규 정적 클래스)가 TitleData를 캐싱해서 `BySkillId`, `RandomByDifficulty` 제공.
- `ProblemDatabase` ScriptableObject는 에디터에서 초기 데이터 관리용으로만 유지(빌드에서 런타임 사용 안 함).

## 6. 엣지 케이스 / 주의점

| 상황 | 처리 방식 |
|------|----------|
| 비밀 키 브루트포스 | CloudScript에서 단기간 연속 실패 시 일시 잠금 (CloudScript 내 PlayFab WriteLog로 기록) |
| WebGL에서 키보드 입력 | TMP_InputField는 WebGL에서 정상 동작. 마스킹(contentType = Password) 필수 |
| CloudScript 호출 실패 | 타임아웃·네트워크 오류 시 StatusText에 에러 표시, 패널 닫히지 않음 |
| ProblemService 캐시 갱신 | D탭에서 문제 수정 후 `ProblemService.InvalidateCache(charId)` 호출해 다음 전투 전 재로드 |
| 회수로 덱 공백 | 전투 중인 캐릭터를 회수할 경우 덱에서 제거 로직 함께 처리 |
| TitleData 용량 | PlayFab TitleData 키 하나당 최대 10KB. 문제가 많아지면 캐릭터별 분할 키로 이미 대응됨. 초과 시 추가 분할 필요 |
| 문제 id 충돌 | 추가 시 UUID 또는 타임스탬프 기반 id 자동 생성 (CloudScript 내) |
| IsAdmin 역직렬화 | 현재 `LoadAdminFlag()`가 제거되므로, 기존 코드에서 호출하는 곳이 있다면 모두 제거 확인 |

## 7. 검증 방법

| 검증 항목 | 기대 결과 |
|----------|----------|
| 설정 메뉴 → 관리자 로그인 → 올바른 키 입력 | AdminPanel 4탭으로 열림 |
| 잘못된 키 입력 | "잘못된 키입니다" 메시지, 패널 열리지 않음 |
| 게임 재시작 | IsAdmin 초기화, 관리자 패널 닫혀 있음 |
| WebGL 빌드에서 설정 메뉴 접근 | 관리자 로그인 버튼 보임, 키 입력 가능 |
| A탭: 재화 직접 입력 지급 | 입력한 금액만큼 Wallet 증가, 저장 완료 |
| A탭: 레벨 강제 지정 | 해당 캐릭터 레벨 즉시 변경 |
