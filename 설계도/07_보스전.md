# 07. 보스전 — 역사적 과학자 보스

---

## 세계관 맥락

- 랜드마크(유클리드 대성당, 힐베르트의 무한 호텔 등) 진입 → 던전 → 최종 층 보스(역사적 과학자).
- 보스는 해당 학문 분야의 수호자·지배자이며, **과학자 정체성 × 그 던전 특화 능력셋**으로 구성된다.
  - 같은 과학자가 여러 던전에 등장 가능 (예: 뉴턴 = 만유인력 던전 / 미적분 던전 — BossDef 따로).
- **이 설계도는 전투·아레나·데이터 구조를 다룬다.** 던전 진입·층 전환·보상은 02 던전 템플릿 소관.
- **1차 구현 범위**: 보스 공통 **템플릿** + 레퍼런스 보스 **유클리드** (아틀란티스 수학 대륙, 유클리드 대성당).

---

## 1. 목표 / 완료 정의

1. 랜드마크(유클리드 대성당) 진입 → 던전 → 보스 룸 → **등장 배너+대사** → 텔레그래프 회피 전투 → HP 임계값마다 **페이즈 전환 + 노출형 약점 윈도** → **처치 대사** → `BossMission` 완료 → 02 던전 클리어 흐름 연결의 **한 사이클이 동작**한다.
2. `BossDef` ScriptableObject + 패턴 모듈 에셋 배치만으로 새 보스를 만들 수 있는 **데이터 주도 템플릿**이 완성된다 (새 보스 추가 시 코드 0줄).
3. 새 패턴 타입 추가 시 `BossPatternModule` 서브클래스 1개만 작성하면 이후 모든 보스가 데이터로 활용 가능.
4. 유클리드 1체가 위 모든 시스템의 **레퍼런스 구현**으로 동작한다.

## 3. 아키텍처

### 3-1. IDamageable (타격 대상 통합)

`PlayerCombat`이 Enemy·Boss·약점 부위를 동일 인터페이스로 처리.

```csharp
public interface IDamageable
{
    void  ReceiveHit(Continent attackerElement, int atk);
    Transform Transform { get; }
    bool  IsDead { get; }
}
```

- `Enemy.cs` — `: IDamageable` 추가. 기존 `ReceiveHit` 로직 **변화 없음**.
- `Boss.cs` — 본체 피격 구현.
- `BossWeakPoint.cs` — 노출 약점 구현.
- `PlayerCombat.DoAttack()` — 기존 `Enemy.All` 루프 뒤:
  - `Boss.Active`(본체) 부채꼴 판정
  - `BossWeakPoint.ExposedInstances`(노출된 약점) 각각 부채꼴 판정

### 3-2. 공통 부품 (모든 보스 공유 — 보스 무관 코드)

| 부품 | 유형 | 역할 |
|------|------|------|
| `Boss` | MonoBehaviour, IDamageable | 본체: HP/피격/사망, `static Boss Active`, `event OnDefeated`, 약점 노출 제어 |

### 3-3. Boss (본체)

```
Boss : MonoBehaviour, IDamageable
├── static Boss Active              — OnEnable 등록 / OnDisable 해제
├── [SerializeField] BossDef _def   — 인스펙터 연결
├── EnemyUnit _unit                 — Start에서 def 값으로 생성
├── BossWeakPoint _weakPoint        — 약점 코어 참조 (같은 GO 또는 자식)
├── HpFraction : float              — _unit.HpFraction 위임
├── ReceiveHit(Continent, int)      — CombatMath로 배율 계산 + DamageNumber 스폰. 사망 시 StartDeath()
├── ReceiveWeakHit(int rawDmg)      — 약점 적중 시 BossWeakPoint → Boss로 보너스 배율 적용 피해
├── event Action OnDefeated         — 처치 연출 끝나고 발행 (BossMission이 구독)
└── Coroutine StartDeath()          — 패턴 정지 → 잔여 Telegraph 정리 → 처치 배너 → OnDefeated 발행
```

**비주얼**: `BossVisualBuilder` 정적 헬퍼(보스 룸 에디터 빌더에서 호출)로 보스별 외형 구성.  
유클리드 = 흰 대리석 대형 캡슐 + 황금 라인 + 떠다니는 기하 도형(컴퍼스·직각자 큐브).

### 3-4. BossPatternModule (패턴 SO 모듈)

```csharp
public abstract class BossPatternModule : ScriptableObject
{
    // 수치 파라미터는 서브클래스에서 직접 SerializeField로 보유
    public abstract IEnumerator Execute(BossContext ctx);
}
```

**1차 구현 모듈 4종**:

| 클래스 | 에셋 예시 | 직렬화 파라미터 |
|--------|---------|--------------|
| `LineBeamPattern` | `Euclid_LineBeam.asset` | warnTime, length, width, damage |
| `CircleZonePattern` | `Euclid_CircleZone.asset` | warnTime, radius, damage |
| `ParallelLasersPattern` | `Euclid_ParallelLasers.asset` | warnTime, separation, length, damage |
| `PolygonBarragePattern` | `Euclid_Barrage.asset` | projectileCount, speed, damage, lifetime |

각 모듈은 `Execute`에서 `Telegraph.Spawn*` / `BossProjectile.Spawn` 으로 프리미티브를 조립하고 완료 시 `yield break`.

**확장 경로** (코드 1파일 → 새 능력 전 보스 공유):
- 소환: `SummonMinionsPattern.cs` — `EnemySpawner` 확장 또는 새 스폰 포인트.
- 지속 해저드: `PersistentHazardPattern.cs` — 수명이 긴 `Telegraph` 변형.
- 지형 변경: `ArenaModPattern.cs` — 발판 온/오프 등.

### 3-5. BossBrain (상태기계, generic)

보스별 서브클래스를 만들지 않는다. `BossDef` 데이터로만 동작.

```
BossBrain : MonoBehaviour
├── [SerializeField] Boss _boss
├── int _phase = 0                  — 0-based
├── bool _active = false            — 연출·처치 중 패턴 중단
│
├── void StartBattle()              — 외부(BossRoomTrigger)에서 호출
│     → BossHealthBar.Show
│     → StartCoroutine(PlayIntro)
│     → StartCoroutine(RunPatternLoop)
│
├── IEnumerator PlayIntro()         — 등장 배너(_def.introLine, 2초)
│
├── void Update()                   — HpFraction 임계값 감시
│     → 교차 시 StartCoroutine(PhaseTransition)
│
├── IEnumerator PhaseTransition()
│     → _active = true
│     → 전환 배너(_def.phaseLines[_phase], 1.5초)
│     → _phase++
│     → exposeWeakPointOnEnter → BossWeakPoint.SetExposed(exposeDuration)
│     → _active = false
│
├── IEnumerator RunPatternLoop()
│     → loop:
│         yield until !_active && !UIManager.IsAnyPanelOpen
│         BossPatternModule mod = 현재 페이즈 모듈 배열에서 순환/랜덤 선택
│         BossContext ctx = new(boss, boss.transform, player.transform)
│         yield return StartCoroutine(mod.Execute(ctx))
│         yield new WaitForSeconds(currentPhaseDef.patternInterval)
│
└── void Cleanup()                  — 처치 시: StopAllCoroutines + 잔여 Telegraph 일괄 Destroy
```

### 3-6. 약점 2종

#### ① 속성 약점 (기존 CombatMath)
`BossDef.weakness : Continent`. `Boss.ReceiveHit`에서 `CombatMath.ComputeDamage(atk, def, isWeak)` 그대로.

#### ② 노출형 타격지점 약점 (BossWeakPoint, 신규)
지정 패턴 완료 / 페이즈 전환 후 `BossBrain`이 `BossWeakPoint.SetExposed(N초)` 호출.

```
BossWeakPoint : MonoBehaviour, IDamageable
├── static List<BossWeakPoint> ExposedInstances  — PlayerCombat 탐색용
├── bool Exposed { get; private set; }
├── SetExposed(float duration)  — Exposed=true, 비주얼(빛나는 코어), 타이머 후 닫힘
├── ReceiveHit(Continent, int)  — Exposed일 때만 → Boss.ReceiveWeakHit(atk * weakMultiplier)
│                                 Exposed=false 시 무반응
└── weakMultiplier (Inspector)  — 기본 3.0f
```

**비주얼**: 코어 Sphere(Emission 노랑→주황), 노출 중 Renderer 활성, 닫히면 비활성.  
**용도**: 속성 약점과 독립. 약점 타이밍을 잡으면 페이즈 가속·보너스 데미지 버스트.

### 3-7. Telegraph / BossProjectile (프리미티브)

**Telegraph** (예고 장판):
```
static Telegraph SpawnLine(Vector3 origin, Vector3 direction, float length, float warnTime, int damage)
static Telegraph SpawnCircle(Vector3 center, float radius, float warnTime, int damage)
흐름:
  Awake → 형태에 맞는 납작 프리미티브 생성 (반투명 빨강)
  코루틴 → warnTime*0.8초 대기 → 플래시(불투명) → warnTime*0.2초 대기
          → Physics.OverlapBox/Sphere → Player 태그 적중 시 Damage()
          → Destroy(gameObject)
```
직선 = 길쭉 Cube (두께 0.15, Y=+0.05). 원 = 납작 Cylinder (Y=+0.05).

**BossProjectile** (이동 투사체):
```
static BossProjectile Spawn(Vector3 pos, Vector3 dir, float speed, int damage, float lifetime)
Update → transform.position += _dir * _speed * Time.deltaTime
OnTriggerEnter(Player 태그) → Damage() → Destroy
수명 초과 → Destroy
```
작은 구 (Scale 0.3), 보스 색 계열.

### 3-10. 연출 흐름 (전체)

```
[보스 룸 진입]
  BossRoomTrigger.OnTriggerEnter → BossBrain.StartBattle()
    → BossHealthBar.Show(보스명, 페이즈 수)
    → PlayIntro() 코루틴 (등장 배너, 2초)
    → RunPatternLoop() 코루틴 시작

[패턴 루프]
  모듈 선택 → Execute(ctx) yield → patternInterval 대기 → 반복
  UIManager.IsAnyPanelOpen 시 루프 내 yield until 해제

[HP 임계값 교차]
  BossBrain.Update() 감지 → PhaseTransition() 코루틴
    → _active = true (패턴 루프 일시정지)
    → 전환 배너(phaseLine, 1.5초)
    → _phase++
    → exposeWeakPointOnEnter == true → BossWeakPoint.SetExposed(N초)
    → _active = false (루프 재개)

[HP = 0]
  Boss.StartDeath() 코루틴
    → BossBrain.Cleanup() (코루틴 정지, Telegraph 일괄 Destroy)
    → 처치 배너(defeatLine, 2초)
    → BossHealthBar.Hide()
    → OnDefeated 발행 → BossMission.CompleteMission() (구독)
    → 02 던전 클리어 흐름
```

## 4. 유클리드 구체 구현 (레퍼런스)

**정체성**: 기하학의 아버지 / 『원론』 / 5공준 수호자. 순수수학파(엘프) 거점 아틀란티스 유클리드 평원의 주인.  
**약점**: 속성 = 논리·집합(Math의 추상적 적 / 튜닝 확정 필요). 타격지점 = 가슴 황금비 코어.

**비주얼 (BossVisualBuilder)**
- 대형 흰 캡슐 (height 2.4, radius 0.6) + 황금 라인 Cube 띠 3개
- 머리 위 회전하는 정삼각형·정사각형·정육각형 미니어처 (크기 0.15–0.3)
- 황금 코어 Sphere (가슴, Emission On/Off = 약점 노출 여부)

**페이즈 구성**

| 페이즈 | HP 범위 | 가용 모듈 에셋 | 약점 노출 |
|--------|---------|--------------|---------|
| 1 | 100–66% | `Euclid_LineBeam` `Euclid_CircleZone` | 없음 |
| 2 | 66–33% | + `Euclid_ParallelLasers` (빈도↑) | 전환 진입 시 5초 |
| 3 | 33–0% | + `Euclid_PolygonBarrage` (N=6→8→12 단계) | 탄막 후 5초 |

**모듈 파라미터 (초안 튜닝값)**

| 에셋 | 주요 수치 |
|------|---------|
| `Euclid_LineBeam` | length 14, width 1.2, warnTime 1.0s, damage 20 |
| `Euclid_CircleZone` | radius 3.5, warnTime 1.2s, damage 25 |
| `Euclid_ParallelLasers` | separation 3.0, length 12, warnTime 0.8s, damage 18 |
| `Euclid_PolygonBarrage` | count 6(P3초반)/12(P3중반), speed 6, damage 15, lifetime 4s |

**페이즈 인터벌**: P1=3.0s, P2=2.5s, P3=2.0s

**대사**
- 등장: "기하학에 왕도는 없다 — 증명할 자격을 보여라."
- P2 전환: "제5공준: 평행선은 영원히 만나지 않는다."
- P3 전환: "원론의 마지막 정리를 받아라."
- 처치: "…완벽한 증명이었다."

## 5. 유클리드 대성당 — 아레나 & 던전 구조

### 5-1. 위치·진입
- **외부 랜드마크**: `AtlantisHubBuilder`가 생성한 유클리드 평원 섬의 유클리드 대성당 오브젝트.
- 대성당 입구에 `Interactable` 부착 → E키 → 던전 씬 전환 (01 씬 전환 인프라).

### 5-2. 던전 구조 (1차 = 단일 루트)

```
[유클리드 대성당 던전]
 Floor 1 — 기하 파수꾼 (KillMission)
 Floor 2 — 체크포인트 / 원형 미로 (MazeMission)
 Floor 3 (Boss) — 유클리드 본인
```

**향후 확장 대비**: `FloorDef`에 `nextFloors : FloorDef[]` 배열(현재 02에서 단일 next) 추가 → 분기 층 지원. 보스룸 복수 시 택1 처치로 `DungeonRunner.OnAnyBossDefeated()` 클리어 판정. 구체 구현은 02 갱신 세션 별도.

## 8. 엣지 케이스 / 주의점

| 상황 | 처리 방식 |
|------|----------|
| 동시 보스 1체 | `Boss.OnEnable` — `Active != null` 이면 경고 로그 + 기존 Active 해제 후 등록 |
| 페이즈 전환 중 피격 | `_active=true`여도 `Boss.ReceiveHit`는 계속 처리. 패턴 실행만 중단 |
| 텔레그래프 발동 전 보스 사망 | `Boss.StartDeath` 초반 `FindObjectsByType<Telegraph>()` 전체 Destroy |
| 약점 노출 중 보스 사망 | StartDeath → BossWeakPoint.SetExposed(0) 강제 종료 |
| BossProjectile 화면 밖 방치 | lifetime(기본 5초) 초과 Destroy |
| WeakPoint ReceiveHit — Exposed=false | 무반응. PlayerCombat 루프에서 `ExposedInstances`만 탐색하므로 중복 판정 없음 |
| 처치 연출 중 BossMission 중복 완료 | BossMission에 `_done bool` — 1회만 CompleteMission() 호출 |
| UIManager 패널 열림 중 패턴 | RunPatternLoop 내 `yield return new WaitUntil(() => !UIManager.IsAnyPanelOpen)` |
| 보스 사망 후 Active 잔류 | `Boss.OnDisable` — `if (Active == this) Active = null` |
| 봉인 문 닫힘 중 전멸·재입장 | PlayerDeath → DungeonRunner.OnPartyWiped → 씬 복귀. 재입장 시 Boss 재스폰(룸 프리팹 재활성) |
| 패턴 모듈 에셋 null | `BossBrain.RunPatternLoop` 진입 시 null 체크 + 경고 로그, 해당 슬롯 스킵 |
