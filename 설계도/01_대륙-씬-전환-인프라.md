# 01. 세계 구조 & 씬 스트리밍

---

## 세계관 맥락

- **메조리아 허브**: 중앙 허브. 6대륙으로 나가는 포탈이 있음. 처음엔 모두 잠금.
- **대륙(과목)**: 물리·화학·생물·지구과학·수학·정보. 각 대륙 안에 국가들이 있음.
- **국가 내부**: 안전지대 (전투 없음, NPC·랜드마크·던전 입구 있음).
- **대륙↔대륙, 국가↔국가 사이**: 오픈필드 (적 출현).
- **던전**: 국가 내부 랜드마크에서 진입. 별도 씬(02번). 이 설계도는 던전 내부를 다루지 않음.

---

## 1. 목표 / 완료 정의

- 영구 부트스트랩 씬 + 플레이어 주변 그리드 셀 **애디티브 로드/언로드**로 끊김 없는 월드 이동이 동작한다.
- 오픈필드 트리거 진입 시 적 스폰, 안전지대 복귀 시 적 디스폰이 동작한다.
- 포탈 해금(오픈필드로 직접 도달) → 이후 포탈 순간이동, 떠난 자리로 복귀가 동작한다.
- 던전 입·퇴장 시 월드↔던전 씬 전환 + 파티·재화·세이브 상태가 유지된다.

## 2. 현재 상태 & 재사용 자산

| 자산 | 경로 | 재사용 방식 |
|------|------|-----------|
| 영속 상태 | `Scripts/Meta/MetaState.cs`, `PlayerRuntime.cs` | static이라 씬 전환에도 유지. 항목만 확장 |
| 부트스트랩 | `GameBootstrap.cs` | 현재 씬마다 `MetaState.Init()` 호출 → **영구 씬으로 이동해 1회만 Init** |
| 적 레지스트리 | `Scripts/Combat/Enemy.cs` (`Enemy.All`) | `OnDisable`에서 자동 제거 → 셀 언로드 시 자연 정리 |
| 적 스포너 | `Scripts/Combat/EnemySpawner.cs` | `ZoneTrigger`에서 오픈필드 진입 시 스폰 호출 |
| 상호작용 | `Scripts/Interaction/Interactable.cs` | 포탈·던전 입구에 부착, E키 이벤트 사용 |
| 저장 | `Backend/MetaSaveService.cs` | `KEY_WORLD` 추가. 기존 Export/LoadState + dict 패턴 그대로 |

## 3. 아키텍처

### 씬 구성

```
[Bootstrap.unity]  ← 항상 떠 있는 영구 씬
  - Player
  - Main Camera
  - HUD Canvas (기존 CombatHud 포함)
  - GameBootstrap        ← 1회 Init (씬마다 재실행 제거)
  - WorldStreamer         ← 그리드 로드/언로드 관리
  - WorldTravelService   ← 순간이동·던전 입퇴장 담당

[Scenes/World/Cell_X_Y.unity]  ← 그리드 셀들, 애디티브 로드
  - 지형, 구조물, NPC
  - ZoneTrigger (Safe / OpenField)
  - EnemySpawner (오픈필드 셀에만)
  - Portal (포탈이 있는 셀에만, Interactable)
  - Dungeon Entrance (랜드마크 셀에만, Interactable)
```

### WorldGridConfig (ScriptableObject)

```
cellSize      : float   — 셀 한 변의 월드 단위 크기 (예: 200f)
loadRadius    : int     — 플레이어 주변 N 셀 반경 로드 (예: 2 → 5×5)
unloadRadius  : int     — loadRadius + 1 (히스테리시스, 팝인 방지)
worldOrigin   : Vector2 — 셀 좌표 계산 기준점
```

좌표 → 셀 변환: `cellId = Vector2Int(Floor(pos.x / cellSize), Floor(pos.z / cellSize))`  
셀 ID → 씬 이름: `"Cell_" + cellId.x + "_" + cellId.y`

### WorldStreamer (MonoBehaviour, 영구 씬)

```
흐름
Update마다
  currentCell = 플레이어 위치 → cellId 변환
  currentCell 변경됐을 때만:
    loadRadius 안의 셀 → 로드 안 된 것만 LoadSceneAsync(Additive)
    unloadRadius 밖의 셀 → UnloadSceneAsync
```

- 포탈 순간이동 직후 / 던전 복귀 직후: 도착 위치 주변 셀 동기 로드 완료까지 플레이어 이동 입력 보류.
- 던전 진입 시: 모든 월드 셀 `UnloadSceneAsync` 완료 후 던전 씬 로드.

### ZoneTrigger (MonoBehaviour, 셀 씬 안에 배치)

```
zoneType : ZoneType  — Safe / OpenField
spawner  : EnemySpawner 참조
```

- `OnTriggerEnter(Player)` → OpenField: `spawner.StartSpawning()`
- `OnTriggerExit(Player)` → OpenField: `spawner.StopAndClear()`
- 여러 OpenField 존을 동시에 걸쳐도 정상 동작해야 함 (참조 카운팅).

### Portal (MonoBehaviour + Interactable, 셀 씬 안에 배치)

```
portalId   : string  — 이 포탈의 고유 ID
targetPos  : Vector3 — 이동 목적지 월드 좌표
isUnlocked : bool    — 초기 false, 잠금 해제 후 true
```

- 플레이어가 포탈 구역에 진입 → `WorldState.UnlockPortal(portalId)` 호출(해금).
- 해금된 포탈 E키 → `WorldTravelService.TeleportTo(targetPos)`.
- 잠긴 포탈은 Interactable 비활성(또는 "아직 해금 안 됨" 메시지).

### WorldTravelService (MonoBehaviour, 영구 씬)

```
TeleportTo(Vector3 dest)
  → PlayerRuntime.LastWorldPos = Player.position   (현재 위치 저장)
  → Player.position = dest
  → WorldStreamer.ForceRefresh()                   (셀 즉시 재계산)
  → MetaSaveService.Save()

EnterDungeon(string dungeonSceneName)
  → PlayerRuntime.LastWorldPos = Player.position
  → WorldStreamer.UnloadAll()
  → SceneManager.LoadSceneAsync(dungeonSceneName, Additive)
  → MetaSaveService.Save()

ExitDungeon()
  → SceneManager.UnloadSceneAsync(dungeonSceneName)
  → Player.position = PlayerRuntime.LastWorldPos
  → WorldStreamer.ForceRefresh()
  → MetaSaveService.Save()
```

### 저장: WorldState

```
WorldState
├── HashSet<string> unlockedPortals  — 해금된 포탈 ID 집합
└── Vector3         lastWorldPos     — 마지막 월드 좌표 (던전·로그아웃 복귀용)

WorldStateData (Serializable)
├── string[] unlockedPortals
├── float    lastX, lastY, lastZ
```

- `MetaState`에 `WorldState` 등록.
- `MetaSaveService`에 `KEY_WORLD = "worldState"` 추가, 기존 Save/Load dict 패턴 그대로.

## 6. 엣지 케이스 / 주의점

| 상황 | 처리 방식 |
|------|----------|
| 좌표 정밀도 | 월드가 커지면 원점에서 먼 위치에서 물리·위치 흔들림 발생. 초기엔 맵 규모 제한으로 회피. 나중에 필요 시 Floating Origin(플레이어 원점 재조정) 적용 |
| 셀 경계 팝인 | `unloadRadius = loadRadius + 1` 히스테리시스 유지. 경계선 이동 반복 시 로드/언로드 진동 방지 |
| 언로드 중 전투 | 전투 중인 적이 있는 셀은 언로드 보류(`Enemy.All`에 해당 셀 적 있으면 스킵) |
| 포탈 순간이동 직후 | 도착 셀 로드 완료까지 플레이어 입력·낙하 보류. `WorldStreamer.IsLoading` 플래그로 관리 |
| 던전 진입 시 메모리 | 월드 셀 전부 언로드 완료 후 던전 씬 로드(동시 로드 방지) |
| GameBootstrap 중복 Init | Bootstrap 씬이 항상 떠 있으므로 `MetaState.Init()`은 게임 최초 시작 시 1회만. 씬 로드로 재호출 없음 |
| 세이브 실패 | PlayFab 실패 시 로그 후 진행 계속. 인메모리 상태는 유지 |

## 7. 검증 방법

### 최소 검증 세트

| 검증 항목 | 기대 결과 |
|----------|----------|
| 플레이어 이동 → 셀 경계 통과 | 주변 N×N 셀만 로드, 먼 셀은 언로드 (Hierarchy에서 확인) |
| 오픈필드 트리거 진입 | 적 스폰 시작 |
| 안전지대 복귀 | 적 디스폰, 전투 해제 |
| 포탈 직접 도달 | `WorldState.unlockedPortals`에 ID 추가, 포탈 E키 활성화 |
| 포탈 E키 → 순간이동 | 목적지 좌표로 이동, 직전 위치 `LastWorldPos` 저장 확인 |
| 던전 입장 | 월드 셀 전부 언로드 → 던전 씬 로드 |
| 던전 퇴장 | 던전 씬 언로드 → 랜드마크 직전 위치로 복귀, 주변 셀 재로드 |
| 재접속 | 마지막 저장 월드 좌표에서 시작 |
