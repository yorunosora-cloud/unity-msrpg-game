# 05. 퀘스트 시스템

---

## 역할

| 역할 | 설명 |
|------|------|
| 스토리·목표 제시 | 플레이어에게 다음 행동 방향을 제시 |
| 대화·전투·탐험 연결 | NPC 대화(04), 던전(02), 세계 이동(01)의 이벤트를 목표로 묶음 |
| 06 길드 게시판의 퀘스트 소스 | 길드 패널이 이 시스템의 QuestDef를 목록으로 표시 |
| 호감도·승급 트리거 기반 | 퀘스트 완료를 조건으로 하는 다른 시스템의 선행 기준 |

---

## 1. 목표 / 완료 정의

- `QuestDef` 에셋만 추가하면 새 퀘스트가 동작하는 **템플릿** 구조가 동작한다.
- 수락(대화 / 게시판 / 자동 트리거) → 단계·목표 진행 → 완료(자동 또는 NPC 납품) → 보상 지급 → 저장의 한 사이클이 동작한다.
- 목표 타입 5종(적 처치 / NPC 대화 / 아이템 수집·납품 / 던전 클리어·지역 도달 / 구역 도달)이 각 이벤트 소스와 연동된다.
- 메인 퀘스트 선행 체인, 퀘스트 로그·HUD 트래커, PlayFab 저장·복원이 동작한다.

## 2. 현재 상태 & 재사용 자산

| 자산 | 경로 | 재사용 방식 |
|------|------|-----------|
| QuestService 스텁 | `Scripts/Dialogue/QuestStartAction.cs`, `QuestCompleteAction.cs` | 스텁 → 실제 `QuestService.Start/Complete` 호출로 교체 |
| 대화 조건 스텁 | `Scripts/Dialogue/DialogueCondition.cs` (`QuestActive`, `QuestComplete`) | `QuestService.IsActive/IsComplete` 실제 참조로 교체 |
| 보상 구조 | `Scripts/Dungeon/RewardDef.cs` | `QuestDef.reward`에 그대로 재사용 |
| 보상 지급 API | `Wallet.Add`, `StudyMaterials.Add`, `CombatCharacter.GainExp` | 퀘스트 완료 보상 지급에 재사용 |
| Enemy 사망 이벤트 | `Scripts/Combat/Enemy.cs` | 적 처치 목표 카운트용 보고 지점 삽입 |
| 대화 완료 이벤트 | `Scripts/Dialogue/DialoguePlayer.cs` (`OnDialogueEnd`) | NPC 대화 목표 보고 지점 삽입 |
| 던전 클리어 이벤트 | `Scripts/Dungeon/DungeonRunner.cs` (보스 처치 후) | 던전 클리어 목표 보고 지점 삽입 |
| 구역 도달 | `Scripts/World/ZoneTrigger.cs`, `Portal.cs` | 구역·지역 도달 목표 보고 지점 삽입 |
| 저장 | `Backend/MetaSaveService.cs` | `KEY_QUEST` 추가, 기존 Export/LoadState dict 패턴 |

## 3. 아키텍처

### 데이터 (ScriptableObject / Serializable)

```
QuestDef (ScriptableObject)
├── questId              : string
├── displayName          : string
├── description          : string
├── questType            : QuestType        — Main / Side
├── prerequisiteQuestIds : string[]         — 선행 완료 필요 퀘스트 ID 목록 (Main에서 주로 사용)
├── autoStartCondition   : QuestTrigger     — 자동 수락 조건 (null이면 수동 수락만)
├── completionMode       : CompletionMode   — AutoComplete / TurnInToNpc
├── turnInNpcId          : string           — 납품형일 때 보고할 NPC ID (AutoComplete면 무시)
├── steps                : QuestStep[]      — 순차 단계 목록 (index 0 = 첫 단계)
└── reward               : RewardDef        — 완료 보상 (02의 RewardDef 재사용)

QuestStep (Serializable)
├── stepId       : string
├── description  : string                   — HUD·로그에 표시
└── objectives   : QuestObjective[]         — 이 단계의 목표들 (전부 충족 시 다음 단계)

QuestObjective (Serializable)
├── objectiveType : ObjectiveType
│   — KillEnemy    (적 처치, targetId=enemyId, 빈 문자열=아무 적)
│   — TalkToNpc    (NPC 대화 완료, targetId=npcId)
│   — CollectItem  (아이템 수집, targetId=itemId)
│   — DeliverItem  (아이템 납품 NPC, targetId=npcId)
│   — ClearDungeon (던전 클리어, targetId=dungeonId)
│   — ReachZone    (구역/지역 도달, targetId=zoneId 또는 portalId)
├── targetId      : string   — 빈 문자열이면 타입 일치만으로 카운트
├── requiredCount : int      — 목표 수량 (TalkToNpc·ReachZone은 1 고정)
└── description   : string   — UI 표시 ("늑대 5마리 처치")

QuestTrigger (Serializable — autoStartCondition)
├── triggerType : TriggerType — OnQuestComplete / OnReachZone / OnLevelReach
├── targetId    : string      — 선행 퀘스트 ID / 구역 ID / (레벨은 targetId 미사용)
└── requiredLevel : int       — OnLevelReach 일 때 필요 레벨
```

---

### 런타임 상태

```
QuestProgress (퀘스트 1개 진행 상태)
├── questId      : string
├── status       : QuestStatus   — Active / ReadyToComplete / Completed
├── currentStep  : int           — 현재 진행 중인 단계 인덱스
└── counts       : int[]         — 현재 단계 각 목표의 진행 수치 (objectives 순서와 1:1)

QuestProgressData (Serializable)
├── questId, status, currentStep
└── int[] counts
```

`MetaState.QuestState` — 런타임 저장소

```
QuestState
├── Dictionary<string, QuestProgress> progresses
├── bool   IsActive(questId)     — Active 또는 ReadyToComplete
├── bool   IsComplete(questId)   — Completed
└── QuestProgress GetProgress(questId)
```

---

### QuestEvent (보고 구조체)

각 시스템이 `QuestService.Report`로 보내는 단일 구조체.

```
QuestEvent (Serializable struct)
├── eventType  : ObjectiveType   — 발생한 이벤트 종류
├── targetId   : string          — 처치된 enemyId / 대화한 npcId / 수집한 itemId 등
└── count      : int             — 이 이벤트로 늘어난 수량 (보통 1)
```

---

### QuestService (static 클래스 — 핵심 로직)

```
Start(questId)
  → QuestDef 로드
  → 선행조건 검사 (prerequisiteQuestIds 전부 Completed 여부)
  → 실패 시: 로그 + return false
  → 성공 시: QuestProgress(Active, currentStep=0, counts 초기화) 등록
  → MetaSaveService.Save()

Report(QuestEvent evt)
  → progresses 중 Active 상태인 것만 순회
  → 현재 단계(steps[currentStep])의 objectives 와 evt 매칭:
      (objectiveType == evt.eventType) &&
      (string.IsNullOrEmpty(targetId) || targetId == evt.targetId)
  → 매칭된 목표: counts[i] = Min(counts[i] + evt.count, requiredCount)
  → 현재 단계의 모든 objectives가 count >= requiredCount?
      → AdvanceStep()

AdvanceStep()
  → currentStep + 1
  → 다음 단계 있으면: counts 0 초기화, MetaSaveService.Save()
  → 마지막 단계 완료:
      AutoComplete  → Complete(questId)
      TurnInToNpc   → status = ReadyToComplete (NPC 대화 대기)

Complete(questId)
  → status = Completed
  → RewardDef 보상 지급 (Wallet.Add / StudyMaterials.Add / CombatCharacter.GainExp)
  → 보상 중복 방지: 이미 Completed면 return
  → MetaSaveService.Save()
  → CheckAutoStart()   — 이 완료로 자동 시작될 퀘스트 체크

CheckAutoStart()
  → 모든 QuestDef 중 autoStartCondition != null && 아직 Active/Completed 아닌 것
  → 조건 평가: OnQuestComplete → IsComplete(targetId) / OnReachZone → (최근 도달 zoneId) / OnLevelReach → 파티 평균 레벨
  → 충족 시 Start(questId)
```

**단방향 보고 구조**: 각 시스템이 QuestService를 구독자로 등록하지 않고, 이벤트 발생 시 `QuestService.Report(evt)` 한 줄만 삽입. QuestService가 내부에서 매칭. 결합도 최소.

### 이벤트 보고 삽입 지점

| 이벤트 | 파일 | 삽입 위치 | QuestEvent |
|--------|------|-----------|-----------|
| 적 사망 | `Enemy.cs` | `OnDeath()` 또는 HP=0 처리 후 | `KillEnemy, enemyId, 1` |
| NPC 대화 완료 | `DialoguePlayer.cs` | `EndDialogue()` 직전 | `TalkToNpc, npcId, 1` |
| 아이템 획득 | 인벤토리/Interactable | 아이템 획득 처리 후 | `CollectItem, itemId, 1` |
| 아이템 납품 | 납품 NPC 대화 완료 시 | `EndDialogue()` 또는 `QuestCompleteAction` | `DeliverItem, npcId, 1` |
| 던전 클리어 | `DungeonRunner.cs` | 보스 클리어 → 씬 복귀 직전 | `ClearDungeon, dungeonId, 1` |
| 구역 도달 | `ZoneTrigger.cs`, `Portal.cs` | `OnTriggerEnter(Player)` 또는 해금 시 | `ReachZone, zoneId, 1` |

### 04 대화 시스템 연결 (스텁 실제화)

**QuestStartAction.Execute()**
```
→ QuestService.Start(questId)
→ 실패 시 (선행조건 미충족) 대화 흐름에 영향 없음, 로그만
```

**QuestCompleteAction.Execute()**
```
→ ReadyToComplete 상태인지 확인
→ QuestService.Complete(questId)
```

**DialogueCondition 검사 (QuestActive / QuestComplete)**
```
QuestActive   → MetaState.QuestState.IsActive(targetId)
QuestComplete → MetaState.QuestState.IsComplete(targetId)
```

## 6. 엣지 케이스 / 주의점

| 상황 | 처리 방식 |
|------|----------|
| 인벤토리 시스템 미확정 | `CollectItem`·`DeliverItem` 보고 지점은 인벤토리 구현 후 삽입. 목표 타입 정의는 미리 두되, 보고 코드만 지연 |
| 동일 이벤트가 여러 퀘스트에 매칭 | 전부 갱신 — 의도된 동작. 카운트 중복 없음(각 퀘스트 자신의 counts만 증가) |
| 선행조건 미충족 Start | `return false` + 디버그 로그. 대화 QuestStartAction도 동일 가드 |
| ReadyToComplete 중 재실행 방지 | `Report`에서 ReadyToComplete 상태는 진행 무시 — 이미 목표 달성 완료 |
| 보상 중복 지급 방지 | `Complete()` 진입 시 `status == Completed`이면 즉시 return |
| 단계 전환 중 저장 실패 | 실패 로그 후 진행 계속. 인메모리 상태는 유지, 재접속 시 이전 체크포인트 단계로 돌아갈 수 있음 |
| QuestDef 에셋 없는 ID 참조 | `QuestService.Start()`에서 null 체크 → 경고 로그 + return false |
| HUD 트래커 미추적 상태 | PlayerPrefs 키 없으면 QuestTracker 비표시. 새 퀘스트 수락 시 자동 추적 선택 안 함 — 사용자가 로그에서 수동 선택 |
