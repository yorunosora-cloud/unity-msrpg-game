# 06. 길드 퀘스트 패널 설계도

> 목적: 모험가 길드 건물 E키 → 퀘스트 게시판 패널 오픈. 목록 보기 + 수락만 담당. 완료·보상은 NPC 대화 시스템(04·05)에서 처리.

---

## 1. 목표 / 완료 정의

- 길드 건물에서 E키를 누르면 `GuildPanel`이 열린다.
- 패널에는 `showInGuild: true`인 퀘스트를 **난이도(Easy/Normal/Hard) 탭**별로 표시한다.
- 각 퀘스트 행을 클릭하면 **인라인 아코디언** 으로 설명·목표·보상 미리보기가 펼쳐진다.
- 이미 진행 중이 아닌 퀘스트만 수락 버튼이 활성화된다. 동시 수락은 **5개 한도**.
- 수락 시 `QuestService.Start(questId)` 호출 → **토스트 메시지** + **버튼 "진행 중"으로 상태 변경**.
- 길드 등급·명성 시스템 연동은 현재 없음 (나중에 추가 예정 — 구조만 열어 둠).

---

## 2. 현재 상태 & 재사용 자산

| 자산 | 경로 | 재사용 내용 |
|------|------|------------|
| `MetaPanelController` | `Backend/MetaPanelController.cs` | `OpenGuild()` 메서드 추가. `UIManager.TryOpen()` + `SetActive` 동일 패턴 |
| `UIManager.TryOpen()` | (기존) | 패널 중복 오픈 방지 |
| `QuestDef` | `Scripts/Quest/QuestDef.cs` | `showInGuild`, `difficulty` 필드 추가 (설계도 05 신규 파일) |
| `QuestService.Start()` | `Scripts/Quest/QuestService.cs` | 선행조건·한도 체크 후 Active 등록 |
| `MetaState.QuestState` | `Scripts/Meta/QuestState.cs` | Active 퀘스트 수 조회 (`GetActiveCount()`) |
| `ToastManager` or HUD 공통 토스트 | (기존 혹은 신규 공통 컴포넌트) | 수락 시 짧은 알림 메시지 표시 |

## 3. 아키텍처

### 3-1. QuestDef 추가 필드 (05번 파일 수정)

```csharp
// QuestDef.cs 에 추가
public enum QuestDifficulty { Easy, Normal, Hard }

[Header("길드 게시판")]
public bool showInGuild;             // true 이면 길드 패널에 표시
public QuestDifficulty difficulty;   // Easy / Normal / Hard
```

### 3-2. GuildPanel 구조

```
GuildPanel (GameObject, 기본 비활성)
├── DifficultyTabBar      — Easy / Normal / Hard 탭 버튼 3개
│   └── onTabClick → GuildPanel.OnTabSelected(difficulty)
└── QuestScrollView
    └── Content
        └── GuildQuestRow × N (동적 생성)
            ├── HeaderRow    — 퀘스트명 / 난이도 배지 / 상태 표시
            ├── DetailPanel  — 아코디언 (기본 비활성)
            │   ├── Description
            │   ├── ObjectiveList
            │   └── RewardPreview
            └── AcceptButton — "수락" / "진행 중" / "완료됨"
```

### 3-3. GuildPanel 로직 흐름

```
OnEnable()
  └─ Refresh(): QuestDef 전체 로드 → showInGuild 필터 → 현재 탭 difficulty 필터
                → QuestService 상태 조회 → GuildQuestRow 생성·갱신

OnTabSelected(difficulty)
  └─ Refresh() (탭 전환 시 재필터)

GuildQuestRow.OnHeaderClick()
  └─ DetailPanel.SetActive(!isOpen)   ← 아코디언 토글
     다른 열린 행 닫기 (동시 1개만 펼침)

GuildQuestRow.OnAcceptClick()
  ├─ QuestService.GetActiveCount() >= 5 → 토스트("퀘스트를 5개 이상 진행할 수 없습니다") → return
  ├─ QuestService.Start(questId) 실패(선행조건 미충족) → 토스트(사유) → return
  └─ 성공 → 버튼 텍스트 "진행 중" + 비활성화, 토스트("[퀘스트명] 수락")
```

### 3-4. 버튼 상태 판별

| 조건 | 버튼 텍스트 | 인터랙터블 |
|------|------------|-----------|
| NotStarted, 선행조건 충족, Active 수 < 5 | 수락 | true |
| NotStarted, 선행조건 미충족 | 수락 (잠금) | false |
| Active | 진행 중 | false |
| ReadyToComplete | 완료 보고 (비활) | false |
| Completed | 완료됨 | false |
| Active 수 >= 5 | 수락 (한도) | true (클릭 시 토스트) |

### 3-5. MetaPanelController 추가

```csharp
[SerializeField] GameObject guildPanel;   // 인스펙터 연결

public void OpenGuild()
{
    if (guildPanel == null) return;
    if (!UIManager.TryOpen()) return;
    guildPanel.SetActive(true);
}
```

### 3-6. 길드 건물 Interactable 연결

- 길드 건물 GameObject의 `Interactable.onInteract` → `MetaPanelController.OpenGuild()` 연결
- 기존 `OpenLab()` / `OpenLibrary()` 가 `Interactable.onInteract`로 연결된 것과 동일

## 6. 엣지 케이스 / 주의점

- **Active 5개 한도 체크 위치**: `QuestService.Start()` 내부에서 거부 + 반환값/예외로 알려야 한다. GuildPanel은 그 결과를 받아 토스트를 띄우는 것이지, 직접 `GetActiveCount()`만으로 판단하면 NPC 수락 경로와 이중 체크가 생긴다. `QuestService.Start()`가 `StartResult` (enum: Success / AlreadyActive / LimitReached / PrereqNotMet) 를 반환하도록 05번 구현 시 설계할 것.
- **아코디언 동시 1개**: `GuildPanel.Refresh()` 또는 `OnHeaderClick()`에서 현재 열린 행을 추적해 다른 행 클릭 시 닫는다. `currentOpenRow` 참조 보관.
- **QuestDef 전체 로드 방법**: 05번 구현 시 `QuestDatabase` (모든 QuestDef 에셋을 Resources 또는 Addressables로 관리) 가 있어야 필터링 가능. 없으면 `Resources.LoadAll<QuestDef>("Quests")` 폴백.
- **패널 열릴 때마다 Refresh**: OnEnable에서 호출해 진행 상태가 실시간 반영되도록 한다 (수락 후 재오픈 시 "진행 중" 표시).
- **보상 미리보기**: QuestDef.reward(RewardDef) 기반으로 표시만 하고, 실제 보상 지급은 하지 않는다. (완료 → NPC 대화 경로)
- **길드 등급 확장 여지**: `GuildPanel`에 `requiredGuildRank: int` 필드(QuestDef) 추가 시 버튼 잠금 조건에 편입할 수 있도록 버튼 상태 판별을 메서드로 분리해 둔다.
- **UI 닫힘 시 상태 보존 불필요**: 패널은 SetActive(false) 시 Destroy하지 않고 비활성만. 탭·스크롤 위치는 다음 오픈 시 OnEnable Refresh가 재구성하므로 저장 불필요.

## 7. 검증 방법

1. **기본 흐름**: 길드 건물에 접근 → E키 → GuildPanel 열림 확인
2. **탭 전환**: Easy/Normal/Hard 탭 클릭 시 해당 난이도 퀘스트만 표시
3. **아코디언**: 행 클릭 → 세부 정보 펼침, 다른 행 클릭 → 이전 행 닫히고 새 행 열림
4. **수락 정상**: 수락 버튼 클릭 → 토스트("[퀘스트명] 수락") + 버튼 "진행 중" 비활성화
5. **한도 초과**: 5개 Active 상태에서 추가 수락 시도 → 토스트("5개 이상 진행 불가") + 수락 안 됨
6. **선행조건 미충족**: 미완료 선행 퀘스트가 있는 퀘스트 수락 → 버튼 비활성 또는 토스트
7. **재오픈**: 패널 닫고 다시 열었을 때 수락한 퀘스트가 "진행 중"으로 표시
8. **UIManager 중복 방지**: 다른 패널 열린 상태에서 길드 건물 E키 → GuildPanel 열리지 않음
9. Claude가 PowerShell 스크린샷으로 직접 확인 (메모리 규칙)
