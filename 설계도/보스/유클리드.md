## 1. 정체성 · 세계관 컨텍스트

### 핵심 테마
**"기하학에 왕도는 없다 — 증명할 자격을 보여라."**

기하학의 아버지. 『원론(Elements)』의 저자이자 5공준의 수호자. 유클리드 평원 대성당의 주인으로, 아틀란티스 5대 공리 국가 중 **가장 안정적이고 고전적인** 순수수학의 기반을 지킨다.

### 세계관 근거 (세계관 아이디어-2.txt L228–274)
- 유클리드 평원: 중심 학문 = 기하학, 공리주의, 유클리드 기하학. 순수수학 기반.
- 순수파 엘프: 고립주의. 순수수학파는 유클리드 평원과 논리 고원에서 강함.
- 유클리드 대성당: "완벽한 직선과 원으로 이루어진 거대 건축물."
- 내부 갈등: 순수수학파(고립주의) vs 응용수학파(교류·실용주의).

### 포지션
수학 대륙 `Atlantis.unity` 씬의 **첫 보스**. **벽 같은 보스** — 약점/시너지 파악 없이는 매우 길고 고통스러운 전투. 기하학의 룰을 배워야 이길 수 있는 개념 관문.

### 2-B 미래 (최종 아트 가이드)

- **모델**: 흰 대리석 질감의 키 큰 엘프 현자. 로브 형태.
- **소품**: 부유하는 황금 컴퍼스·직각자(각 패턴 시전 시 해당 도구 강조).
- **왕관**: 정다각형 형태의 기하 왕관 — PolygonBarrage 시전 시 발광·방출.
- **발광**: 황금 라인 띠 → URP Emission 발광. 약점 노출 시 가슴 코어 황금→주황 Emission.
- **셰이딩**: URP 툰 셰이더 + 외곽선 (적용 시점: 맵·캐릭터 아트 완성 단계 — `celshading_mandatory` 메모리 참조).
- **팔레트**: 흰 대리석(#F5F0E8) + 황금(#FFD700) + 수학 대륙 팔레트 (`design_system` 메모리 참조).
- **실루엣**: 키 약 3 Unity units. 상반신 인간형, 도형 왕관으로 머리 볼륨감.

## 3-bis. 보스 모션 · 이동 명세

### 이동 방식: 느린 추적 + 페이즈 순간이동

#### 느린 추적 (BossLocomotion — 신규 코드 필요)
- 패턴 캐스팅 사이(idle 구간) 플레이어를 향해 저속 이동.
- **속도**: ~1.5 m/s (플레이어 속도의 약 1/3 — 잡히진 않지만 공간을 계속 압박).
- 항상 플레이어를 정면으로 향함 (`face = player direction`).
- 패턴 Execute 시작 시 이동 정지, 완료 후 재개.

#### 페이즈 순간이동
- **P2·P3 진입 시**: 중앙 제단 위치(0, 0, 0)로 즉각 텔레포트 → 플레이어 정면 재배치.
- 배너 표시 + 짧은 무적(~1초)과 동기화.
- 텔레포트 VFX: ① 지금 = 없음(즉각 이동). ② 미래 = 도형 분해→재조립 파티클.
- 기둥 꼭짓점 8방향 텔레포트는 향후 확장 항목.

> ⚠ **신규 코드 필요**: `BossLocomotion` MonoBehaviour. 현재 `Boss`/`BossBrain`에 locomotion 없음.
> 후속 세션(Task 12-bis)에서 구현. 이 문서는 동작·파라미터만 명세.

### 지금 (프리미티브) 레이어 모션

사지·시전 애니 **없음**. 프로시저럴만:

| 효과 | 조건 | 구현 방법 |
|------|------|-----------|
| 머리 위 정다각형 자전 | 항상 | `Update()`: `HeadGeo.Rotate(0, rotSpeed * Time.deltaTime, 0)` |
| 자전 가속 | P2 60°/s, P3 120°/s | `BossBrain` 페이즈 진입 시 rotSpeed 갱신 |
| 황금 띠 펄스 | P3 진입 후 | `GoldBand` Emission intensity를 `Mathf.PingPong` |
| 약점 코어 발광 | 약점 노출 시 | `BossWeakPoint.OpenWeakPoint()` → Renderer enabled + Emission ON |

### 미래 (리깅) 레이어 모션
- 아이들 호흡·서 있는 자세
- 패턴별 시그니처 시전 제스처 (§3 테이블 참조)
- 피격·사망 애니
- 페이즈 텔레포트 VFX (도형 분해→재조립)

## 5. 약점 메커니즘

### 가슴 황금비 코어 (BossWeakPoint)
- **위치**: 가슴 전면 (localPos 0, 1.5, 0.62 — Capsule 반지름 밖으로 돌출).
- **노출 타이밍**:
  - P2 진입 즉시 → `exposeWeakPointOnEnter = true`, `weakPointExposeDuration = 5s`
  - P3: PolygonBarrage 패턴 Execute 종료 직후 → BossBrain에서 수동 `SetExposed(5f)` 호출 (07 §4 패턴 훅 참조)
- **시각 피드백**: 노출 = Renderer enable + 황금 Emission 점등. 숨김 = Renderer disable.
- **데미지 배율**: ×3 (`_weakMultiplier = 3`), 속성 무관.
- **코드 경로**: `BossWeakPoint.ReceiveHit` → `Boss.ReceiveWeakHit(bonusDmg)` → `DamageNumber.Spawn(isCrit: true)`.

### weakness 속성 보너스 (Boss.ReceiveHit)
- `weakness = Math`. Math 속성 캐릭터의 일반 공격이 `CombatMath.ComputeDamage`에서 보너스 적용.
- 약점 코어(×3)와는 **독립된 별개 메커니즘**.

## 9. 보상 · 클리어 연동

처치 흐름: `Boss.OnDefeated` → `BossMission.OnBossDefeated` → 던전 클리어 처리.

드랍·보상 세부는 [02_던전-스테이지-템플릿.md](../02_던전-스테이지-템플릿.md) 소관. 이 문서에서는 초안 제안만:
- **경험치**: 별도 (`expReward = 0` — BossDef 기본값, 시스템에서 별도 처리 예정).
- **전리품 후보**: 공준석(기하 정수?), 컴퍼스 유물(장비?).

## 10. 검증 방법

Play 모드에서 스크린샷으로 확인 (`feedback_game_verification` 메모리 참조):

- [ ] **아레나 진입**: BossRoomTrigger 통과 → BossHealthBar 등장 + 배너 introLine 출력.
- [ ] **P1 패턴**: LineBeam(황금 직선 텔레그래프) + CircleZone(원형 텔레그래프) 정상 발동.
- [ ] **약점 코어**: P2 진입 시 가슴 코어 5초 노출 → 공격 시 isCrit DamageNumber ×3.
- [ ] **P2 배너**: HP 66% 돌파 → "공준만으로 충분하다…" 배너 + 중앙 텔레포트.
- [ ] **P3 배너**: HP 33% 돌파 → "원론의 마지막 장이다…" 배너 + PolygonBarrage 추가.
- [ ] **정팔각형 탄막**: PolygonBarrage가 8방향 발사 + 종료 후 코어 재노출.
- [ ] **처치**: HP 0 → defeatLine + BossHealthBar 숨김 + SealDoor 비활성화.
- [ ] **보스 추적**: 패턴 사이 플레이어 방향 저속 이동 확인 (BossLocomotion 구현 후).

## 11. 향후 확장

- **리깅 모션**: 패턴별 시그니처 제스처(§3 미래 컬럼), 아이들·피격·사망 애니, 텔레포트 VFX.
- **셀셰이딩**: URP 툰 셰이더 + 외곽선 적용 (맵·캐릭터 아트 완성 후 — `celshading_mandatory`).
- **PolygonBarrage 변형**: 정6→8→12각 에셋 분기 (P3 진행에 따른 단계 강화).
- **기둥 텔레포트**: P3에서 8기둥 꼭짓점으로 순간이동 + 패턴 기준점 변경.
- **다른 기하학자**: 같은 던전에 다른 BossDef (비유클리드 기하학 보스 등) 추가.
- **미래 아트 레이어 전환**: 미래 모델 완성 후 EuclidSetup 프리미티브 → 아트 에셋으로 교체.
